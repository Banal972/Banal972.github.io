프로그래머스 - 같은 숫자는 싫어

코딩 테스트

2024-08-17

3분

문제

배열 arr가 주어집니다. 배열 arr의 각 원소는 숫자 0부터 9까지로 이루어져 있습니다. 이때, 배열 arr에서 연속적으로 나타나는 숫자는 하나만 남기고 전부 제거하려고 합니다. 단, 제거된 후 남은 수들을 반환할 때는 배열 arr의 원소들의 순서를 유지해야 합니다. 예를 들면,

  • arr = [1, 1, 3, 3, 0, 1, 1] 이면 [1, 3, 0, 1] 을 return 합니다.
  • arr = [4, 4, 4, 3, 3] 이면 [4, 3] 을 return 합니다.

배열 arr에서 연속적으로 나타나는 숫자는 제거하고 남은 수들을 return 하는 solution 함수를 완성해 주세요.

제한사항

  • 배열 arr의 크기 : 1,000,000 이하의 자연수
  • 배열 arr의 원소의 크기 : 0보다 크거나 같고 9보다 작거나 같은 정수

입출력 예

arranswer
[1,1,3,3,0,1,1][1,3,0,1]
[4,4,4,3,3][4,3]

풀이과정

처음에 풀었던 방식

arr.reduce((ac, cur) => {
  if (ac.length > 0 && ac[ac.length - 1] === cur) return ac;
  return [...ac, cur]; 
}, []);

해당 방식 같은 경우 테스트는 모두 통과를 했습니다.
하지만 효율성에서는 테스트를 통과하지 못했는데
이것은 시간 복잡도에 문제가 있었습니다. 빅오 표기법을 아신다면 O(1), O(n) 이런것을 아실텐데
reduce와 스프레드 연산자를 동시에 사용 했기 때문에 O(n2)가 되어버려 속도가 느려 효율성에 통과를 하지 못했었습니다.

시간 복잡도를 고쳐본 방식

function solution(arr){
    const answer = [];

    for (let i = 0; i < arr.length; i++) {
      // O(n)
      if (answer.length > 0 && answer[answer.length - 1] === arr[i]) continue;
      answer.push(arr[i]);
    }
    
    return answer;
}

저는 O(n2)가 아닌 O(n)이 될 수 있도록 for문으로 방식을 수정했고 효율성에 통과 할 수 있게 되었습니다.
